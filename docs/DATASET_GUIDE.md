# Space Weather Dataset Guide for Deep Learning / 딥러닝을 위한 우주환경 데이터셋 가이드

This document describes the structure, variable definitions, and usage of the space weather dataset generated by the `setup-sw-db` project.

이 문서는 `setup-sw-db` 프로젝트에서 생성된 우주환경 데이터셋의 구조, 변수 의미, 사용 방법을 설명합니다.

---

## 1. Dataset Overview / 데이터셋 개요

Time series data of solar wind parameters and geomagnetic indices aggregated at 30-minute intervals.

30분 간격으로 집계된 태양풍 파라미터와 지자기 지수의 시계열 데이터입니다.

- **Time resolution / 시간 해상도**: 30 min (48 records per day / 하루 48개 레코드)
- **Number of variables / 변수 수**: 23 (excluding datetime / datetime 제외)
- **Data period / 데이터 기간**: 2010 – present (OMNI 1-min data availability range / OMNI 1분 데이터 가용 범위)
- **Sources / 출처**: NASA OMNI (1-min resolution / 1분 해상도) + GFZ Potsdam HPo (30-min resolution / 30분 해상도)

---

## 2. Variable Descriptions / 변수 설명

### 2.1 Solar Wind Parameters (OMNI, 21 columns) / 태양풍 파라미터 (OMNI, 21개 컬럼)

For each variable, the **mean (avg), minimum (min), and maximum (max)** are computed from 1-minute data within each 30-minute window `[t, t+30min)`.

각 변수는 30분 윈도우 `[t, t+30min)` 내의 1분 데이터로부터 **평균(avg), 최솟값(min), 최댓값(max)**을 계산합니다.

| Variable Group / 변수 그룹 | Column Name / 컬럼명 | Description / 설명 | Unit / 단위 | Typical Range / 일반적 범위 |
|-----------|--------|------|------|------------|
| **Solar Wind Speed (V) / 태양풍 속도 (V)** | `v_avg`, `v_min`, `v_max` | Solar wind plasma bulk speed / 태양풍 플라즈마 유속 | km/s | 250 ~ 800 |
| **Proton Density (Np) / 양성자 밀도 (Np)** | `np_avg`, `np_min`, `np_max` | Solar wind proton number density / 태양풍 양성자 수밀도 | n/cm³ | 0.1 ~ 50 |
| **Proton Temperature (T) / 양성자 온도 (T)** | `t_avg`, `t_min`, `t_max` | Solar wind proton temperature / 태양풍 양성자 온도 | K (Kelvin / 켈빈) | 10⁴ ~ 10⁶ |
| **IMF Bx** | `bx_avg`, `bx_min`, `bx_max` | IMF X component (GSE coordinates) / 행성간 자기장 X 성분 (GSE 좌표) | nT | -20 ~ +20 |
| **IMF By** | `by_avg`, `by_min`, `by_max` | IMF Y component (GSM coordinates) / 행성간 자기장 Y 성분 (GSM 좌표) | nT | -20 ~ +20 |
| **IMF Bz** | `bz_avg`, `bz_min`, `bz_max` | IMF Z component (GSM coordinates) / 행성간 자기장 Z 성분 (GSM 좌표) | nT | -20 ~ +20 |
| **IMF Magnitude (Bt) / IMF 크기 (Bt)** | `bt_avg`, `bt_min`, `bt_max` | Total IMF magnitude / 행성간 자기장 전체 크기 | nT | 0 ~ 50 |

### 2.2 Geomagnetic Indices (HPo, 2 columns) / 지자기 지수 (HPo, 2개 컬럼)

HPo data is already at 30-minute intervals, so it is matched directly without aggregation.

HPo 데이터는 이미 30분 간격이므로 집계 없이 직접 매칭됩니다.

| Column Name / 컬럼명 | Description / 설명 | Type / 타입 | Range / 범위 |
|--------|------|------|------|
| `ap30` | 30-min equivalent amplitude geomagnetic index / 30분 등가진폭 지자기 지수 | SMALLINT (integer / 정수) | 0 ~ 400 |
| `hp30` | 30-min Hp geomagnetic index / 30분 Hp 지자기 지수 | REAL (float / 실수) | 0 ~ 9+ |

## 3. Coordinate System Reference / 좌표계 참고

- **GSE (Geocentric Solar Ecliptic)**: X-axis points toward the Sun, Z-axis toward the ecliptic north pole. Used for Bx.
- **GSE (지심태양황도 좌표)**: X축이 태양 방향, Z축이 황도면 북극. Bx에 사용.
- **GSM (Geocentric Solar Magnetospheric)**: X-axis points toward the Sun, Z-axis aligned with Earth's magnetic dipole. Used for By, Bz.
- **GSM (지심태양자기권 좌표)**: X축이 태양 방향, Z축이 지구 자기 쌍극자 방향. By, Bz에 사용.
- By and Bz use the GSM coordinate system because GSM is more suitable for analyzing correlations with geomagnetic activity.
- By, Bz가 GSM 좌표인 이유: 지자기 활동과의 상관관계 분석에 GSM이 더 적합하기 때문입니다.

## 4. Physical Meaning of Variables / 변수별 물리적 의미

| Variable / 변수 | Physical Meaning / 물리적 의미 | Relevance to Geomagnetic Storms / 지자기 폭풍과의 관련성 |
|------|-----------|---------------------|
| **V** (Solar Wind Speed / 태양풍 속도) | Speed at which solar wind reaches Earth. High-speed solar wind (>500 km/s) originates from coronal holes. / 태양풍이 지구에 도달하는 속도. 고속 태양풍(>500 km/s)은 코로나 홀에서 기원 | Higher speed → increased geomagnetic disturbance / 높은 속도 → 지자기 교란 증가 |
| **Np** (Proton Density / 양성자 밀도) | Number of protons in the solar wind. Surges upon CME shock wave arrival. / 태양풍 내 양성자 수. CME 충격파 도달 시 급증 | Density surge → increased dynamic pressure → magnetosphere compression / 밀도 급증 → 동적 압력 증가 → 자기권 압축 |
| **T** (Temperature / 온도) | Thermal energy of solar wind protons. Rises sharply behind shock waves. / 태양풍 양성자의 열 에너지. 충격파 후방에서 급상승 | Temperature spike signals passage of an interplanetary shock wave / 온도 급등은 행성간 충격파 통과의 신호 |
| **Bz** (North-South Magnetic Field / 남북 자기장) | **The most important variable.** Negative Bz (southward) triggers magnetic reconnection with Earth's field. / **가장 중요한 변수**. 음의 Bz(남향)는 지구 자기장과 재결합(reconnection) 유발 | Bz < 0 → magnetic energy injection → triggers geomagnetic storms / Bz < 0 → 자기 에너지 유입 → 지자기 폭풍 유발 |
| **Bt** (Magnetic Field Magnitude / 자기장 크기) | Total IMF strength. Increases inside CME flux ropes. / IMF 전체 세기. CME 내부 자속관에서 증가 | Large Bt + southward Bz = strong storm conditions / 큰 Bt + 남향 Bz = 강한 폭풍 조건 |
| **ap30** | Linear measure of global geomagnetic activity level / 전지구 지자기 활동 수준의 선형 척도 | Direct measurement of geomagnetic disturbance / 지자기 교란의 직접적 측정값 |
| **hp30** | 30-min version of the Kp index (quasi-logarithmic scale) / Kp 지수의 30분 버전 (반로그 척도) | hp30 ≥ 5: geomagnetic storm / 지자기 폭풍, ≥ 7: strong storm / 강한 폭풍 |

## 5. Data Format / 데이터 형식

### 5.1 Individual Event CSV / 개별 이벤트 CSV

Each CSV file represents a single event window.

각 CSV 파일은 하나의 이벤트 윈도우를 나타냅니다.

```
datetime,v_avg,v_min,v_max,np_avg,np_min,np_max,t_avg,t_min,t_max,bx_avg,bx_min,bx_max,by_avg,by_min,by_max,bz_avg,bz_min,bz_max,bt_avg,bt_min,bt_max,ap30,hp30
2021-01-05 00:00:00,389.2,385.0,393.1,4.2,3.8,4.6,52340.0,48200.0,55600.0,-2.1,-3.4,-0.8,1.5,0.2,2.8,-0.3,-1.2,0.6,4.1,3.5,4.8,6,2.0
2021-01-05 00:30:00,391.5,387.3,395.8,...
...
```

### 5.2 Event Window Structure / 이벤트 윈도우 구성

Events are extracted by specifying the number of days before and after a reference time **T**.

기준 시각 **T**를 중심으로 전후 일수를 지정하여 추출합니다.

| Parameter / 파라미터 | Description / 설명 | Default / 기본값 |
|----------|------|--------|
| T | Reference time (e.g., event occurrence time) / 기준 시각 (이벤트 발생 시점 등) | Required / 필수 |
| B | Days before T / T 이전 일수 | 5 days / 5일 |
| A | Days after T / T 이후 일수 | 3 days / 3일 |

- **Start / 시작**: `T - B days`
- **End / 종료**: `T + A days - 30 min`
- **Number of rows / 행 수**: `(B + A) x 48`

**Example / 예시**: T = 2021-01-10 00:00, B=5, A=3
- Range / 범위: 2021-01-05 00:00 ~ 2021-01-12 23:30
- Number of rows / 행 수: 8 x 48 = **384 rows / 384행**

### 5.3 File Naming Convention / 파일명 규칙

During batch extraction, each file is named by the timestamp of the reference time T.

배치 추출 시 각 파일은 기준 시각 T의 타임스탬프로 명명됩니다.

```
YYYYMMDDHHMMSS.csv
e.g. / 예: 20210110000000.csv  →  T = 2021-01-10 00:00:00
```

## 6. Missing Data / 결측값 (Missing Data)

### Causes / 발생 원인

- Observation gaps in the OMNI source data (e.g., satellite data reception failure)
- OMNI 원본 데이터의 관측 공백 (위성 데이터 수신 실패 등)
- Unpublished intervals in the HPo index
- HPo 지수의 미발행 구간

### Representation / 표현 방식

- In CSV files, missing values appear as empty strings.
- CSV에서는 빈 값 (empty string)으로 표현
- When loaded with Pandas, they are automatically converted to `NaN`.
- Pandas로 읽으면 `NaN`으로 자동 처리

### NaN Handling in Batch Extraction / 배치 추출 시 NaN 처리

- **In batch extraction (`extract_sw_events.py`), events containing any NaN are automatically excluded.**
- **배치 추출(`extract_sw_events.py`)에서는 NaN이 하나라도 포함된 이벤트는 자동으로 제외됩니다.**
- This means all provided CSV files are guaranteed to contain complete data (no NaN).
- 즉, 제공된 CSV 파일들은 모두 완전한 데이터(NaN 없음)를 보장합니다.

### NaN Handling in Single Extraction / 단일 추출 시 NaN 처리

- In single extraction (`build_sw_30min.py extract`), NaN filtering is not performed, so missing values may be present.
- 단일 추출(`build_sw_30min.py extract`)에서는 NaN 필터링을 하지 않으므로, 결측값이 포함될 수 있습니다.
- Manual inspection and handling are required.
- 직접 확인 및 처리가 필요합니다.

## 7. Data Loading Examples (Python) / 데이터 로드 예시 (Python)

### Loading a Single File / 단일 파일 로드

```python
import pandas as pd

df = pd.read_csv("20210110000000.csv", parse_dates=["datetime"])
print(df.shape)       # (384, 24)
print(df.dtypes)
print(df.describe())
```

### Batch File Loading / 배치 파일 로드

```python
import glob
import numpy as np
import pandas as pd

files = sorted(glob.glob("/path/to/events/*.csv"))
events = []

for f in files:
    df = pd.read_csv(f, parse_dates=["datetime"])
    # Exclude datetime column, extract numerical data only
    # datetime 열 제외, 수치 데이터만 추출
    values = df.drop(columns=["datetime"]).values  # shape: (384, 23)
    events.append(values)

# Full dataset / 전체 데이터셋
dataset = np.stack(events)  # shape: (N_events, 384, 23)
print(f"Number of events / 이벤트 수: {dataset.shape[0]}")
print(f"Sequence length / 시퀀스 길이: {dataset.shape[1]}")
print(f"Number of variables / 변수 수: {dataset.shape[2]}")
```

### Per-Variable Normalization Example / 변수별 정규화 예시

```python
# Per-variable (per-channel) normalization
# 변수별 (채널별) 정규화
mean = dataset.mean(axis=(0, 1))   # shape: (23,)
std = dataset.std(axis=(0, 1))     # shape: (23,)
normalized = (dataset - mean) / (std + 1e-8)

# Or Min-Max normalization / 또는 Min-Max 정규화
vmin = dataset.min(axis=(0, 1))
vmax = dataset.max(axis=(0, 1))
scaled = (dataset - vmin) / (vmax - vmin + 1e-8)
```

## 10. Important Notes / 주의사항

1. **All timestamps are in UTC.** They are recorded in UTC, not KST (Korea Standard Time).

   **모든 시각은 UTC입니다.** KST(한국 표준시)가 아닌 UTC 기준으로 기록되어 있습니다.

2. **Pay attention to the sign of Bz.** Negative Bz (southward) is the key driver of geomagnetic storms. Since the sign carries physical meaning, be cautious when applying absolute value transformations.

   **Bz의 부호에 주의하세요.** 음의 Bz(남향)가 지자기 폭풍의 핵심 드라이버입니다. 부호가 물리적 의미를 가지므로 절댓값 변환 시 주의가 필요합니다.

3. **Variable scales differ significantly.** Temperature (T) ranges from 10⁴ to 10⁶, while magnetic field (B) is on the order of ±20 nT. Using the data without normalization may cause training issues.

   **변수 스케일 차이가 큽니다.** 온도(T)는 10⁴~10⁶ 범위인 반면, 자기장(B)은 ±20 nT 수준입니다. 정규화 없이 사용 시 모델 학습에 문제가 될 수 있습니다.

4. **avg/min/max triplet structure**: Each physical variable is represented by 3 columns. They capture variability within 30 minutes, so you can use only avg or all three depending on your needs.

   **avg/min/max 트리플렛 구조**: 각 물리 변수가 3개 컬럼으로 표현됩니다. 30분 내의 변동성 정보를 담고 있으므로, 필요에 따라 avg만 사용하거나 전체를 사용할 수 있습니다.

5. **Relationship between ap30 and hp30**: Both represent geomagnetic activity, but ap30 uses a linear scale while hp30 uses a quasi-logarithmic scale. Choose one or use both depending on your application.

   **ap30과 hp30의 관계**: 둘 다 지자기 활동을 나타내지만, ap30은 선형 척도이고 hp30은 반로그 척도(quasi-logarithmic)입니다. 용도에 따라 선택하거나 둘 다 사용할 수 있습니다.

6. **Batch extraction results contain no NaN**, but the number of extractable events may decrease significantly during certain periods (e.g., periods with many data gaps).

   **배치 추출 결과에는 NaN이 없지만**, 특정 기간(예: 데이터 갭이 많은 시기)에서는 추출 가능한 이벤트 수가 크게 줄어들 수 있습니다.
