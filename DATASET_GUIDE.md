# Space Weather Dataset Guide for Deep Learning

이 문서는 `setup-sw-db` 프로젝트에서 생성된 우주환경 데이터셋의 구조, 변수 의미, 사용 방법을 설명합니다.

---

## 1. 데이터셋 개요

30분 간격으로 집계된 태양풍 파라미터와 지자기 지수의 시계열 데이터입니다.

- **시간 해상도**: 30분 (하루 48개 레코드)
- **변수 수**: 23개 (datetime 제외)
- **데이터 기간**: 2010년 ~ 현재 (OMNI 1분 데이터 가용 범위)
- **출처**: NASA OMNI (1분 해상도) + GFZ Potsdam HPo (30분 해상도)

---

## 2. 변수 설명

### 2.1 태양풍 파라미터 (OMNI, 21개 컬럼)

각 변수는 30분 윈도우 `[t, t+30min)` 내의 1분 데이터로부터 **평균(avg), 최솟값(min), 최댓값(max)**을 계산합니다.

| 변수 그룹 | 컬럼명 | 설명 | 단위 | 일반적 범위 |
|-----------|--------|------|------|------------|
| **태양풍 속도 (V)** | `v_avg`, `v_min`, `v_max` | 태양풍 플라즈마 유속 | km/s | 250 ~ 800 |
| **양성자 밀도 (Np)** | `np_avg`, `np_min`, `np_max` | 태양풍 양성자 수밀도 | n/cm³ | 0.1 ~ 50 |
| **양성자 온도 (T)** | `t_avg`, `t_min`, `t_max` | 태양풍 양성자 온도 | K (켈빈) | 10⁴ ~ 10⁶ |
| **IMF Bx** | `bx_avg`, `bx_min`, `bx_max` | 행성간 자기장 X 성분 (GSE 좌표) | nT | -20 ~ +20 |
| **IMF By** | `by_avg`, `by_min`, `by_max` | 행성간 자기장 Y 성분 (GSM 좌표) | nT | -20 ~ +20 |
| **IMF Bz** | `bz_avg`, `bz_min`, `bz_max` | 행성간 자기장 Z 성분 (GSM 좌표) | nT | -20 ~ +20 |
| **IMF 크기 (Bt)** | `bt_avg`, `bt_min`, `bt_max` | 행성간 자기장 전체 크기 | nT | 0 ~ 50 |

### 2.2 지자기 지수 (HPo, 2개 컬럼)

HPo 데이터는 이미 30분 간격이므로 집계 없이 직접 매칭됩니다.

| 컬럼명 | 설명 | 타입 | 범위 |
|--------|------|------|------|
| `ap30` | 30분 등가진폭 지자기 지수 | SMALLINT (정수) | 0 ~ 400 |
| `hp30` | 30분 Hp 지자기 지수 | REAL (실수) | 0 ~ 9+ |

---

## 3. 좌표계 참고

- **GSE (Geocentric Solar Ecliptic)**: X축이 태양 방향, Z축이 황도면 북극. Bx에 사용.
- **GSM (Geocentric Solar Magnetospheric)**: X축이 태양 방향, Z축이 지구 자기 쌍극자 방향. By, Bz에 사용.
- By, Bz가 GSM 좌표인 이유: 지자기 활동과의 상관관계 분석에 GSM이 더 적합하기 때문입니다.

---

## 4. 변수별 물리적 의미

| 변수 | 물리적 의미 | 지자기 폭풍과의 관련성 |
|------|-----------|---------------------|
| **V** (태양풍 속도) | 태양풍이 지구에 도달하는 속도. 고속 태양풍(>500 km/s)은 코로나 홀에서 기원 | 높은 속도 → 지자기 교란 증가 |
| **Np** (양성자 밀도) | 태양풍 내 양성자 수. CME 충격파 도달 시 급증 | 밀도 급증 → 동적 압력 증가 → 자기권 압축 |
| **T** (온도) | 태양풍 양성자의 열 에너지. 충격파 후방에서 급상승 | 온도 급등은 행성간 충격파 통과의 신호 |
| **Bz** (남북 자기장) | **가장 중요한 변수**. 음의 Bz(남향)는 지구 자기장과 재결합(reconnection) 유발 | Bz < 0 → 자기 에너지 유입 → 지자기 폭풍 유발 |
| **Bt** (자기장 크기) | IMF 전체 세기. CME 내부 자속관에서 증가 | 큰 Bt + 남향 Bz = 강한 폭풍 조건 |
| **ap30** | 전지구 지자기 활동 수준의 선형 척도 | 지자기 교란의 직접적 측정값 |
| **hp30** | Kp 지수의 30분 버전 (반로그 척도) | hp30 ≥ 5: 지자기 폭풍, ≥ 7: 강한 폭풍 |

## 5. 데이터 형식

### 5.1 개별 이벤트 CSV

각 CSV 파일은 하나의 이벤트 윈도우를 나타냅니다.

```
datetime,v_avg,v_min,v_max,np_avg,np_min,np_max,t_avg,t_min,t_max,bx_avg,bx_min,bx_max,by_avg,by_min,by_max,bz_avg,bz_min,bz_max,bt_avg,bt_min,bt_max,ap30,hp30
2021-01-05 00:00:00,389.2,385.0,393.1,4.2,3.8,4.6,52340.0,48200.0,55600.0,-2.1,-3.4,-0.8,1.5,0.2,2.8,-0.3,-1.2,0.6,4.1,3.5,4.8,6,2.0
2021-01-05 00:30:00,391.5,387.3,395.8,...
...
```

### 5.2 이벤트 윈도우 구성

기준 시각 **T**를 중심으로 전후 일수를 지정하여 추출합니다.

| 파라미터 | 설명 | 기본값 |
|----------|------|--------|
| T | 기준 시각 (이벤트 발생 시점 등) | 필수 |
| B | T 이전 일수 | 5일 |
| A | T 이후 일수 | 3일 |

- **시작**: `T - B일`
- **종료**: `T + A일 - 30분`
- **행 수**: `(B + A) x 48`

**예시**: T = 2021-01-10 00:00, B=5, A=3
- 범위: 2021-01-05 00:00 ~ 2021-01-12 23:30
- 행 수: 8 x 48 = **384행**

### 5.3 파일명 규칙

배치 추출 시 각 파일은 기준 시각 T의 타임스탬프로 명명됩니다.

```
YYYYMMDDHHMMSS.csv
예: 20210110000000.csv  →  T = 2021-01-10 00:00:00
```

## 6. 결측값 (Missing Data)

### 발생 원인
- OMNI 원본 데이터의 관측 공백 (위성 데이터 수신 실패 등)
- HPo 지수의 미발행 구간

### 표현 방식
- CSV에서는 빈 값 (empty string)으로 표현
- Pandas로 읽으면 `NaN`으로 자동 처리

### 배치 추출 시 NaN 처리
- **배치 추출(`extract_sw_events.py`)에서는 NaN이 하나라도 포함된 이벤트는 자동으로 제외됩니다.**
- 즉, 제공된 CSV 파일들은 모두 완전한 데이터(NaN 없음)를 보장합니다.

### 단일 추출 시 NaN 처리
- 단일 추출(`build_sw_30min.py extract`)에서는 NaN 필터링을 하지 않으므로, 결측값이 포함될 수 있습니다.
- 직접 확인 및 처리가 필요합니다.

## 7. 데이터 로드 예시 (Python)

### 단일 파일 로드

```python
import pandas as pd

df = pd.read_csv("20210110000000.csv", parse_dates=["datetime"])
print(df.shape)       # (384, 24)
print(df.dtypes)
print(df.describe())
```

### 배치 파일 로드

```python
import glob
import numpy as np
import pandas as pd

files = sorted(glob.glob("/path/to/events/*.csv"))
events = []

for f in files:
    df = pd.read_csv(f, parse_dates=["datetime"])
    # datetime 열 제외, 수치 데이터만 추출
    values = df.drop(columns=["datetime"]).values  # shape: (384, 23)
    events.append(values)

# 전체 데이터셋
dataset = np.stack(events)  # shape: (N_events, 384, 23)
print(f"이벤트 수: {dataset.shape[0]}")
print(f"시퀀스 길이: {dataset.shape[1]}")
print(f"변수 수: {dataset.shape[2]}")
```

### 변수별 정규화 예시

```python
# 변수별 (채널별) 정규화
mean = dataset.mean(axis=(0, 1))   # shape: (23,)
std = dataset.std(axis=(0, 1))     # shape: (23,)
normalized = (dataset - mean) / (std + 1e-8)

# 또는 Min-Max 정규화
vmin = dataset.min(axis=(0, 1))
vmax = dataset.max(axis=(0, 1))
scaled = (dataset - vmin) / (vmax - vmin + 1e-8)
```

## 10. 주의사항

1. **모든 시각은 UTC입니다.** KST(한국 표준시)가 아닌 UTC 기준으로 기록되어 있습니다.

2. **Bz의 부호에 주의하세요.** 음의 Bz(남향)가 지자기 폭풍의 핵심 드라이버입니다. 부호가 물리적 의미를 가지므로 절댓값 변환 시 주의가 필요합니다.

3. **변수 스케일 차이가 큽니다.** 온도(T)는 10⁴~10⁶ 범위인 반면, 자기장(B)은 ±20 nT 수준입니다. 정규화 없이 사용 시 모델 학습에 문제가 될 수 있습니다.

4. **avg/min/max 트리플렛 구조**: 각 물리 변수가 3개 컬럼으로 표현됩니다. 30분 내의 변동성 정보를 담고 있으므로, 필요에 따라 avg만 사용하거나 전체를 사용할 수 있습니다.

5. **ap30과 hp30의 관계**: 둘 다 지자기 활동을 나타내지만, ap30은 선형 척도이고 hp30은 반로그 척도(quasi-logarithmic)입니다. 용도에 따라 선택하거나 둘 다 사용할 수 있습니다.

6. **배치 추출 결과에는 NaN이 없지만**, 특정 기간(예: 데이터 갭이 많은 시기)에서는 추출 가능한 이벤트 수가 크게 줄어들 수 있습니다.
